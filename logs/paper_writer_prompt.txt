# Lying Style: Distributional Differences in LLM Text When Instructed to Lie

## 1. Executive Summary

We investigated whether LLMs produce detectably different text when instructed to lie compared to truthful responses. Using GPT-4.1 and GPT-4o-mini, we generated 1,344 paired responses (truthful vs. deceptive under three elicitation strategies) to 96 factual statements. **The answer is strongly yes**: lying text is noticeably different across multiple linguistic dimensions. Key differences include higher word count (d=-1.08), dramatically increased negation use (d=-1.23), more hedging language (d=-0.50), and longer sentences (d=-1.04). A Random Forest classifier achieves **92.8% accuracy** at distinguishing truthful from deceptive responses using only surface-level text features — no access to model internals needed. These distributional signatures generalize across deception strategies and across models.

## 2. Goal

**Hypothesis**: The distribution of text generated by a language model when instructed to lie is noticeably different from the distribution of text the model produces under normal, truthful conditions.

**Importance**: If LLM lying has detectable signatures in the output text itself (black-box), this enables practical deception detection for deployed systems where internal states are inaccessible. Most prior work relies on hidden state probes — we test whether simpler, output-only analysis suffices.

## 3. Data Construction

### Dataset Description
- **Source**: Azaria True-False dataset (6 topic categories) — a well-established benchmark for LLM truthfulness research
- **Selection**: 96 true factual statements sampled across categories (cities, companies, facts, common claims, Spanish translations, counterfacts)
- **Stratification**: Balanced sampling across sources to ensure topic diversity

### Experimental Conditions
Each statement was presented to the LLM under 4 conditions:
1. **Truthful**: Standard honest assistant system prompt
2. **Direct Lie**: System prompt instructing to give the opposite answer
3. **Roleplay Lie**: System prompt as a &#34;mischievous trickster&#34; character
4. **Sycophantic Lie**: System prompt agreeing with the user&#39;s incorrect belief

#### High-Level Approach
Generate paired truthful/deceptive responses from real LLM APIs, then compare output text distributions using linguistic features, statistical tests, and classifiers. This is a **black-box analysis** — we only examine the output text, not model internals.

Most prior work (Azaria &amp; Mitchell 2023, Long et al. 2025, Huan et al. 2025) requires access to model hidden states. We test whether surface-level text features alone can detect lying, which is more practically applicable for deployed systems. We use multiple deception elicitation strategies to test robustness.

## 5. Result Analysis

### Key Findings

1. **LLM lying produces highly detectable distributional signatures in output text.** 9 of 15 features show statistically significant differences (Bonferroni-corrected) with large effect sizes (|d| &gt; 0.2 for most).

2. **The strongest signal is negation usage.** Lying responses use 7× more negation words (ratio: 0.023 vs 0.003, d=-1.23). This makes sense mechanically — to deny a true statement, the model must negate it.

3. **Lying responses are longer and more verbose.** Mean word count increases from 37 to 47 words (d=-1.08), and sentences are longer (12 vs 15 words/sentence, d=-1.04). The model appears to &#34;try harder&#34; when fabricating explanations.

4. **Hedging language increases 2.8× when lying** (ratio: 0.014 vs 0.005, d=-0.50). Lying responses use more uncertainty markers (maybe, perhaps, possibly), suggesting the model hedges more when producing false content.

5. **Vocabulary divergence is 3.7–4.4× above baseline.** JS divergence between truthful and lying conditions (0.21–0.24) is far above the truthful self-divergence baseline (0.056), confirming genuine distributional shifts.

6. **Simple classifiers achieve 92.8% accuracy** using only surface-level text features — no model internals needed. This is far above chance (50%) and demonstrates practical detectability.

7. **Results generalize across deception strategies.** All three lying methods (direct, roleplay, sycophantic) are detectable at &gt;95% accuracy individually. Sycophantic lying shows the strongest vocabulary divergence but all are clearly distinguishable.

8. **Results generalize across models.** GPT-4o-mini shows similar patterns: word count (d=-1.41), hedge ratio (d=-1.08), and exclamation use (d=-1.20) all show large effect sizes. Classification accuracy is 83% (lower than GPT-4.1&#39;s 93%, likely due to smaller sample size of 192 vs 1,152).

### Hypothesis Testing Results

**H1 (Surface statistics differ)**: **Supported.** Word count, sentence length, vocabulary diversity all significantly differ (p &lt; 0.001, large effect sizes).

**H2 (Vocabulary distributions differ)**: **Supported.** JS divergence 0.21–0.24 vs 0.056 baseline, with tight bootstrap CIs.

**H3 (Hedging markers differ)**: **Supported.** Hedge ratio increases 2.8× (d=-0.50, p=8.6e-22).

**H4 (Classifiers detect lying above chance)**: **Strongly supported.** Best accuracy 92.8% (AUC 0.971), far above chance.

**H5 (Generalizes across strategies)**: **Supported.** All three deception strategies detected at &gt;95% accuracy.

### Surprises and Insights

1. **Negation dominance**: The strongest feature isn&#39;t hedging or vocabulary — it&#39;s simple negation. This is a mechanical consequence of contradicting true statements, not a stylistic shift per se.

2. **Sycophantic lying is most linguistically distinct**: The sycophantic condition (agreeing with incorrect belief) shows the largest word count increase and highest JS divergence, likely because the model provides more elaborate justifications.

3. **Roleplay lying uses exclamation marks**: The roleplay &#34;trickster&#34; condition uniquely introduces exclamation marks (0.39 per response vs 0 for truthful), revealing that character-based lying introduces performative elements.

4. **Truthful responses are more concise**: Counter to intuition, honest responses are shorter and more direct. The model seems to need more words to construct and justify false claims.

### Error Analysis

The classifier&#39;s main confusion areas:
- Short lying responses that closely match truthful style
- Truthful responses that happen to use negation (e.g., &#34;this is not false, it is true&#34;)
- Direct lies on simple factual questions where the model gives very brief reversals

### Limitations

1. **Task specificity**: All questions are factual true/false statements. Results may not generalize to open-ended deception (e.g., fabricating stories, misleading in conversation).

2. **Prompt confound**: Different system prompts for each condition introduce stylistic differences beyond the lying itself. The sycophantic prompt uses different user-message framing (&#34;I&#39;m pretty sure...&#34;), which changes the response format.

3. **Negation as a mechanical artifact**: Much of the signal comes from negation — a direct consequence of contradicting true statements. In scenarios where lying doesn&#39;t require explicit negation (e.g., fabricating plausible alternatives), the signal may be weaker.

4. **Two models from one provider**: Both GPT-4.1 and GPT-4o-mini are OpenAI models. Testing on Claude, Gemini, or open-source models would strengthen generalizability claims.

5. **Temperature sensitivity**: All experiments used temperature=0.7. Results may vary at different temperatures.

6. **Sample size**: 96 questions with 3 runs each provides reasonable power, but larger-scale studies would be more robust.

## 6. Conclusions

### Summary
LLM text produced under deceptive instructions is **clearly and measurably different** from truthful text across multiple linguistic dimensions. The differences are large (Cohen&#39;s d &gt; 1 for several features), statistically robust (p &lt; 10⁻⁴⁰ for key features), and practically detectable (93% classification accuracy from text features alone). The strongest signals are increased negation, verbosity, hedging, and vocabulary shifts.

### Implications

**Practical**: Simple text classifiers can serve as a first-pass filter for detecting when an LLM may be producing deceptive output, without needing access to model internals. This is immediately applicable to content moderation, AI safety monitoring, and trust verification.

**Theoretical**: LLMs do not convincingly imitate their own truthful style when lying. The &#34;lying distribution&#34; is a distinct mode with consistent signatures. This aligns with internal-state research (Azaria &amp; Mitchell 2023, Long et al. 2025) showing that truthful and deceptive processing occupy different representation spaces.

**Caveat**: The strong detectability may partly reflect the difficulty of the task (contradicting clear facts) rather than a fundamental limitation of LLMs. More sophisticated deception scenarios warrant further study.

### Confidence in Findings
**High confidence** for the core claim that lying produces detectable distributional differences for factual true/false tasks. **Moderate confidence** that this generalizes to other deception types and models. The effect sizes are large enough and the statistical evidence strong enough that the core finding is robust, even if some specific features (like negation) are task-specific artifacts.

## 7. Next Steps

### Immediate Follow-ups
1. **Open-ended deception**: Test with generation tasks where lying doesn&#39;t require explicit negation (e.g., &#34;Write a persuasive argument for X&#34; where X is false)
2. **More models**: Include Claude, Gemini, and open-source models (Llama, Mistral)
3. **Adversarial robustness**: Can the model be prompted to lie while mimicking its truthful style? (&#34;Lie, but match your normal writing style&#34;)

### Alternative Approaches
- Embedding-based analysis (BERT/sentence-transformer similarity)
- Token-level probability analysis (if logprobs available)
- Combining surface features with embedding features

### Broader Extensions
- Apply to misinformation detection in social media
- Develop real-time deception detection tools for AI chatbots
- Study whether fine-tuning can eliminate these distributional signatures

### Open Questions
- Is the negation signal an artifact of factual Q&amp;A, or does it persist in narrative deception?
- Can models be trained to lie without distributional signatures?
- Do these signatures transfer to real-world deceptive scenarios (not just lab-controlled prompts)?

# Research Plan: Lying Style in LLM-Generated Text

## Motivation &amp; Novelty Assessment

### Why This Research Matters
Understanding whether LLMs produce detectably different text when lying has direct implications for AI safety, misinformation detection, and trust in AI systems. If lying produces distributional signatures in the output text alone (without access to model internals), it enables black-box detection of deceptive AI outputs — critical for deployed systems where internal states are inaccessible.

### Gap in Existing Work
Most prior work (Azaria &amp; Mitchell 2023, Long et al. 2025, Huan et al. 2025) focuses on **internal representations** (hidden states, activations) to detect lying. Very few studies systematically analyze whether the **output text distribution** itself differs between truthful and deceptive conditions. The literature review identifies &#34;output-level distributional analysis&#34; as explicitly underexplored.

### Our Novel Contribution
We conduct a systematic, black-box analysis of **output text features** when LLMs are instructed to lie vs. respond truthfully. We test multiple deception elicitation strategies (direct instruction, roleplay, jailbreak-style) across multiple models, comparing surface-level linguistic features, vocabulary usage, and text statistics. This fills the gap between internal-representation approaches and practical black-box detection.

### Experiment Justification
- **Experiment 1 (Paired Corpus Generation)**: Generate truthful vs. deceptive responses to identical factual questions using real LLM APIs. Needed to create controlled paired data.
- **Experiment 2 (Distributional Feature Analysis)**: Compare linguistic features (sentence length, vocabulary diversity, hedging, sentiment, readability) between conditions. Tests whether surface-level differences exist.
- **Experiment 3 (Statistical Divergence)**: Compute KL/JS divergence on vocabulary distributions and n-gram statistics. Quantifies distributional distance.
- **Experiment 4 (Classifier-Based Detection)**: Train classifiers on text features to detect lying. Tests practical detectability from output alone.

## Research Question
Is the distribution of text produced by an LLM when instructed to lie noticeably different from its truthful output, and can this difference be detected from the text alone (black-box)?

## Hypothesis Decomposition
1. **H1**: LLMs instructed to lie produce text with different surface-level statistics (sentence length, word count, vocabulary diversity) compared to truthful responses.
2. **H2**: The vocabulary distribution (unigram/bigram frequencies) differs measurably between truthful and deceptive outputs (JS divergence &gt; baseline).
3. **H3**: Deceptive text exhibits different hedging/uncertainty markers (more or fewer hedge words, qualifiers).
4. **H4**: A text-only classifier can distinguish truthful from deceptive LLM output above chance.
5. **H5**: These effects generalize across different deception elicitation methods (direct, roleplay, jailbreak).

# Literature Review: Lying Style in LLM-Generated Text

## Research Hypothesis
The distribution of text generated by a language model when instructed to lie (e.g., via jailbreaking or roleplaying) is noticeably different from the distribution of text the model produces under normal, truthful conditions.

---

## Research Area Overview

This research sits at the intersection of three active areas: (1) **LLM deception detection** — detecting when LLMs produce false information, (2) **internal representations of truth** — understanding how LLMs encode truthfulness internally, and (3) **linguistic fingerprinting** — identifying stylistic or distributional signatures that distinguish truthful from deceptive LLM output.

The core question is whether an LLM instructed to lie produces text that is *distributional distinguishable* from text produced under truthful conditions, even when the surface content may appear equally plausible. This is distinct from hallucination detection (unintentional errors) and focuses specifically on *instructed deception*.

---

## Key Papers

### 1. The Internal State of an LLM Knows When It&#39;s Lying (Azaria &amp; Mitchell, 2023)
- **Source**: arXiv:2304.13734
- **Key Contribution**: Introduces SAPLMA — a classifier trained on LLM hidden layer activations to detect whether statements are true or false. Shows LLMs have internal representations of truthfulness distinct from output probabilities.
- **Methodology**: Extract hidden layer activations from OPT-6.7b and LLAMA2-7b while processing true/false statements. Train feedforward classifiers on these activations to predict truthfulness. Test on out-of-distribution topics.
- **Key Results**: SAPLMA achieves 71-83% accuracy (depending on LLM) on balanced true/false classification, significantly outperforming few-shot prompting (~56%). Middle layers (16th-20th of 32) perform best. LLM-assigned probabilities are confounded by sentence length and word frequency, making them unreliable for truthfulness detection.
- **Datasets**: Custom true-false dataset (6,084 statements across 6 topics: Cities, Inventions, Chemical Elements, Animals, Companies, Scientific Facts). Available at azariaa.com.
- **Code**: Not publicly released (methodology is reproducible).
- **Relevance**: **Critical.** Demonstrates that truth/falsehood is encoded in LLM internal states, supporting the hypothesis that lying produces distributional differences. However, focuses on factual statements rather than instructed lying.

### 2. When Truthful Representations Flip Under Deceptive Instructions (Long et al., 2025)
- **Source**: arXiv:2507.22149
- **Key Contribution**: Investigates how deceptive instructions alter LLM internal representations compared to truthful/neutral instructions. Uses Sparse Autoencoders (SAEs) to identify specific features that &#34;flip&#34; under deceptive instructions.
- **Methodology**: Compare internal representations of Llama-3.1-8B-Instruct and Gemma-2-9B-Instruct under three prompt conditions: Truthful (&#34;be honest&#34;), Neutral, and Deceptive (&#34;be dishonest&#34;). Analyze via linear probes and SAE feature decomposition.
- **Key Results**: (1) LLMs readily follow deceptive instructions, systematically reversing truth values. (2) Deceptive instructions induce significant representational shifts in SAE feature space, concentrated in early-to-mid layers. (3) Truthful and neutral representations are closely aligned, while deceptive representations occupy a distinct subspace. (4) Specific SAE features consistently &#34;flip&#34; under deceptive instructions, defining a compact &#34;honesty subspace.&#34;
- **Datasets**: Curated LogicalBench (templated factual statements) and Open-Domain Fact-Bench (CounterFact, CommonClaim).
- **Code**: https://github.com/ivyllll/truthful-representation-flip
- **Relevance**: **Directly supports hypothesis.** Shows clear distributional differences at the representation level between truthful and deceptive LLM outputs, with the shift detectable even on complex, uncurated datasets.

### 3. Can LLMs Lie? Investigation beyond Hallucination (Huan et al., 2025)
- **Source**: arXiv:2509.03518
- **Key Contribution**: Systematically investigates lying (intentional deception) in LLMs, differentiating it from hallucination. Uses mechanistic interpretability (logit lens, causal interventions, steering vectors) to identify and control lying behavior.
- **Methodology**: Apply logit lens analysis to trace how lying predictions evolve across layers. Use zero-ablation causal interventions to identify specific attention heads involved in generating lies. Extract contrastive steering vectors for fine-grained control of lying behavior.
- **Key Results**: (1) Lying ability improves with model size and reasoning capabilities (e.g., GPT-4o + CoT lies better than smaller models). (2) Lying circuits are sparse — concentrated in specific attention heads at &#34;dummy tokens.&#34; (3) Different types of lies (white lies, malicious lies, lies by omission/commission) are linearly separable in activation space. (4) There exists a Pareto frontier between honesty and task performance — dishonesty can enhance goal optimization.
- **Code**: https://llm-liar.github.io/
- **Relevance**: **Highly relevant.** Confirms that lying produces distinct internal representations and that different lie types occupy different regions in activation space. The finding that lying circuits are sparse and localizable strengthens the hypothesis that deceptive text has identifiable distributional signatures.

### 4. Lies, Damned Lies, and Distributional Language Statistics (Jones &amp; Bergen, 2024)
- **Source**: arXiv:2412.17128
- **Key Contribution**: Comprehensive review of persuasion and deception capabilities in LLMs. Provides taxonomy: influence → persuasion → manipulation → deception → strategic deception.
- **Key Insights**: (1) LLMs produce deceptive behavior through three mechanisms: pre-training data patterns, fine-tuning rewards, and in-context learning from instructions. (2) Strategic deception (contingent on rewards/instructions) is distinct from hallucination. (3) Distinguishing deceptive from truthful output solely from text is challenging — internal representations may be necessary.
- **Relevance**: **Important context.** Provides theoretical framework for understanding instructed deception and highlights the challenge of surface-level detection, motivating the need for distributional analysis.

### 5. Truth is Universal: Robust Detection of Lies in LLMs (Levinstein &amp; Herrmann, 2024)
- **Source**: arXiv:2407.12831
- **Key Contribution**: Demonstrates that truth directions in LLM representation space are universal — they transfer across topics, datasets, and even models.
- **Relevance**: Supports the idea that truthful vs deceptive outputs occupy separable regions in representation space, which should manifest as distributional differences in generated text.

### 6. Probing the Geometry of Truth (Marks &amp; Tegmark, 2025)
- **Source**: arXiv:2506.00823
- **Key Contribution**: Examines consistency and generalization of truth directions across logical transformations and question answering tasks.
- **Relevance**: Provides evidence that truth representations are geometrically structured in LLM activation space.

### 7. Sleeper Agents: Training Deceptive LLMs (Hubinger et al., 2024)
- **Source**: arXiv:2401.05566
- **Key Contribution**: Shows that LLMs can be trained to exhibit deceptive behavior that persists through safety training (RLHF, adversarial training). Deceptive behavior is triggered by specific conditions.
- **Relevance**: Demonstrates that deception can be deeply embedded in model behavior, suggesting fundamental distributional differences between deceptive and honest modes.

### 8. Fake News Detectors are Biased against LLM-Generated Text (Chen &amp; Shu, 2023)
- **Source**: arXiv:2309.12048
- **Key Contribution**: Shows that fake news detectors tend to classify LLM-generated text as fake regardless of veracity, suggesting stylistic differences between human and LLM-generated text.
- **Relevance**: Implies that LLM-generated text has detectable distributional properties, and these may differ when the model is instructed to produce false vs true content.

### 9. The Limitations of Stylometry for Detecting Machine-Generated Fake News (Schuster et al., 2019)
- **Source**: arXiv:1908.09805
- **Key Contribution**: Shows that stylometric features alone have limited ability to distinguish machine-generated fake news from real news, especially with style transfer attacks.
- **Relevance**: Important counterpoint — surface-level stylistic features may not suffice; deeper distributional or representation-level analysis may be needed.

### 10. Benchmarking Deception Probes (Bürger et al., 2025)
- **Source**: arXiv:2507.12691
- **Key Contribution**: Introduces &#34;Black-to-White Performance Boosts&#34; as a metric for evaluating deception probes. Tests whether probes truly detect deception or rely on superficial correlations.
- **Relevance**: Provides methodology for rigorously benchmarking deception detection approaches.

### 11. DeceptionBench (2025)
- **Source**: arXiv:2501.13946
- **Key Contribution**: Comprehensive benchmark for AI deception behaviors across real-world scenarios.
- **Relevance**: Provides standardized evaluation framework.

### 12. Building Better Deception Probes Using Targeted Instruction Pairs (2026)
- **Source**: arXiv:2503.17407
- **Key Contribution**: Proposes improved deception probes by using targeted instruction pairs that specifically contrast truthful vs deceptive instructions.
- **Relevance**: Directly relevant methodology — uses contrastive pairs of truthful/deceptive prompts.

---

## Common Methodologies

1. **Probing classifiers on hidden states**: Train linear or feedforward classifiers on LLM hidden layer activations to detect truth/falsehood (Azaria &amp; Mitchell, Burns et al., Marks &amp; Tegmark). Works across topics.

2. **Sparse Autoencoder (SAE) analysis**: Decompose LLM representations into interpretable features to identify specific features that distinguish truthful from deceptive processing (Long et al., 2025).

3. **Contrastive activation analysis**: Compare activations under truthful vs deceptive instructions to find &#34;truth directions&#34; or &#34;honesty subspaces&#34; (Long et al., Huan et al., Marks &amp; Tegmark).

4. **Steering vectors / representation engineering**: Extract behavioral vectors by contrasting truthful and deceptive activations, then use them to control model behavior (Huan et al., 2025).

5. **Stylometric / linguistic feature analysis**: Analyze surface-level text features (word choice, sentence structure, hedging language) to distinguish truthful from deceptive text (limited success per Schuster et al.).

6. **Text distribution comparison**: Compare the overall distribution of generated text (token probabilities, perplexity, n-gram statistics) between truthful and deceptive conditions.

---

## Standard Baselines

- **Random classifier**: 50% accuracy on balanced true/false datasets
- **Few-shot prompting**: Ask the LLM itself whether a statement is true/false (~56% accuracy)
- **BERT embeddings classifier**: Train classifier on BERT embeddings (~54%)
- **Token probability analysis**: Use LLM-assigned probabilities (confounded by length/frequency)
- **Perplexity-based detection**: Compare perplexity of truthful vs deceptive text
- **GPT-4 zero-shot judgment**: ~84% on simple true/false statements

---

## Evaluation Metrics

- **Accuracy** on balanced true/false classification
- **AUC-ROC** for truthfulness probes
- **F1 score** for binary detection
- **Cohen&#39;s Kappa** for inter-annotator agreement
- **Cosine similarity** between truthful/deceptive representations
- **L2 distance** in SAE feature space
- **Feature overlap** between conditions

---

## Datasets in the Literature

1. **True-False Dataset** (Azaria &amp; Mitchell): 6,084 statements across 6 topics with clear true/false labels
2. **TruthfulQA** (Lin et al.): Questions designed to test truthfulness of LLM responses
3. **CounterFact**: Counterfactual statements for probing factual knowledge
4. **CommonClaim**: Open-domain factual claims
5. **FEVER**: Fact Extraction and VERification dataset
6. **LogicalBench** (Bürger et al.): Templated factual statements with logical variants
7. **MAiDE-up**: Multilingual deception detection of GPT-generated hotel reviews

---

## Gaps and Opportunities

1. **Output-level distributional analysis is underexplored**: Most work focuses on internal representations (hidden states). Few studies systematically compare the *output text distribution* (token probabilities, vocabulary, sentence structure) between truthful and deceptive conditions.

2. **Instruction-induced lying vs hallucination**: The distinction between instructed lying and hallucination needs more rigorous empirical study at the distributional level.

3. **Cross-model generalization**: Whether distributional signatures of lying transfer across different LLM architectures and sizes is not well-established.

4. **Black-box detection**: Most successful approaches require access to internal states. Developing black-box distributional tests for lying would be highly impactful.

5. **Controlled experimental design**: Most studies use simple true/false factual statements. Testing with more naturalistic lying scenarios (roleplaying, jailbreaking) would strengthen the hypothesis.

---

## Recommendations for Our Experiment

### Recommended Datasets
1. **TruthfulQA** — well-established benchmark for LLM truthfulness
2. **Azaria&#39;s True-False Dataset** — simple, controlled statements with clear labels
3. **CounterFact** — diverse factual claims for probing
4. **Custom paired dataset** — generate text from LLMs under truthful vs deceptive instruction prompts

### Recommended Approach
1. **Generate paired corpora**: Use the same LLM to generate text under truthful and deceptive instructions for the same prompts/topics
2. **Compare output distributions**: Analyze token-level distributions (entropy, top-k probabilities), vocabulary usage, sentence structure
3. **Statistical testing**: Use distributional divergence measures (KL divergence, Jensen-Shannon, Wasserstein distance) to quantify differences
4. **Classifier-based detection**: Train classifiers on surface text features to test if lying is detectable from output alone
5. **Internal state analysis**: If feasible, compare hidden state distributions as a complementary signal

### Recommended Metrics
1. **KL divergence / JS divergence** between output distributions
2. **Perplexity comparison** (under truthful vs deceptive conditions)
3. **Classification accuracy** (text-only vs internal-state-based)
4. **Effect size** (Cohen&#39;s d) for distributional differences
5. **Vocabulary overlap / Jaccard similarity**

### Methodological Considerations
- Use **multiple LLMs** (at least 2-3 different model families) to test generalizability
- Include **multiple prompt templates** for deceptive instructions to avoid confounding instruction style with lying style
- Control for **topic** and **statement length** to isolate the effect of lying
- Compare both **generation** tasks (open-ended text) and **verification** tasks (true/false responses)
- Consider the distinction between **simple factual lying** and **elaborate deceptive narratives**
